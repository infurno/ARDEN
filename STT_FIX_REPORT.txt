# STT CUDA Out of Memory Error - Fix Applied

**Date:** 2026-01-03  
**Issue:** Speech-to-text failing with CUDA OOM error  
**Status:** ✅ FIXED

## Problem
When sending voice messages via Telegram, Whisper failed with:
```
torch.OutOfMemoryError: CUDA out of memory. 
Tried to allocate 102.00 MiB. GPU has only 107.94 MiB free.
Process 268001 has 9.32 GiB memory in use (qwen2.5-coder:14b)
```

## Root Cause
- qwen2.5-coder:14b (Ollama) uses ~11 GB GPU memory
- Whisper tried to load onto same GPU
- RTX 5070 only has 12 GB total - not enough for both

## Solution
Modified `api/services/stt.js` to force Whisper to use CPU:
```javascript
// Added --device cpu flag
const command = `"${whisperPath}" "${filePath}" --model ${model} --language ${language} --device cpu --output_format txt --output_dir /tmp`;
```

## Trade-offs
✓ **Pros:**
- No more CUDA OOM errors
- qwen2.5-coder keeps full GPU access (fast responses)
- Automatic fallback to OpenAI API if CPU fails

⚠️ **Cons:**
- Whisper transcription slightly slower on CPU (2-5s vs 1-2s)
- Still acceptable for voice messages

## Alternative Solutions Considered
1. **Use smaller AI model** - Would reduce response quality
2. **Use OpenAI Whisper API** - Costs money (~$0.006/minute)
3. **Dynamic GPU allocation** - Complex, not worth the effort

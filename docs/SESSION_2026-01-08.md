## 🎯 Major Accomplishments

### 1. Fixed Notes in Subdirectories ✅
**Problem:** Notes stored in subfolders (e.g., `~/Notes/FDCS-CH5/deployment.md`) couldn't be opened from the web interface.

**Root Causes:**
- Backend search API used `path.basename()` which stripped directory paths
- Frontend used `escapeHtml()` in onclick handlers, corrupting forward slashes
- Only root directory was being scanned

**Solutions Implemented:**
- **Backend (`api/routes/notes.js`):**
  - Added `findMarkdownFiles()` function for recursive directory traversal
  - Updated `sanitizeFilename()` to allow subdirectory paths while preventing traversal attacks
  - Modified GET `/api/notes` endpoint to return relative paths
  - Fixed GET `/api/notes/search` endpoint to include full subdirectory paths
  
- **Frontend (`web/assets/js/notes.js`):**
  - Added `escapeJs()` function for proper JavaScript string escaping
  - Updated onclick handlers to use `escapeJs()` instead of `escapeHtml()`
  
**Commits:**
- `e635e98` - Add subdirectory support to notes API

- `4090fb9` - Fix search results not including subdirectory paths

### 2. Fixed Split View in Note Editor ✅
**Problem:** Split view displayed editor and preview stacked vertically (top/bottom) instead of horizontally (left/right).

**Root Cause:**
- Parent container `#editorContent` wasn't set to `display: flex`
- No `flex-direction: row` specified
- Missing CSS styling for preview pane

**Solutions Implemented:**
- **JavaScript (`web/assets/js/notes.js`):**
  - Added `editorContent.style.display = 'flex'`
  - Added `editorContent.style.flexDirection = 'row'`
  - Enabled GitHub Flavored Markdown (GFM) in marked.js
  
- **CSS (`web/notes.html`):**

  - Table striping with alternating row colors
